## 🎯 한 줄 요약

CS 분야의 **트렌딩(유행성) 수준**을, 논문 데이터로 **3단계(Low/Mid/High)로 분류**하는 모델을 완성했다.

---

## 🧭 최종 설계

### 타겟 (예측 대상)

- **`Avg_MA_7D`** (카테고리별 7일 이동평균 논문 수)
- 분위수로 3등분 → **Low / Mid / High** (클래스 균형)

### 입력 특징 (클린 7개)

- `Category_Covariance_norm` — 분야 내 정규화한 공분산 (가장 중요)
- `Days_Since_Update` — 최신성
- `Paper_Age_Years` — 나이(통제변수)
- `DOI_avg_cite_rate` — 시간보정 출판사 영향력
- `DOI_is_published` — 정식출판 여부
- `Title_Char_Count`, `Title_Word_Count` — 제목 길이

---

## 🧹 마무리에서 정리한 것

### 과감히 제거한 특징

- 🔴 **누수**: `CitationCount`, `Citation_Rate_Annual`, `Citation_Percentile`, `DOI_avg_citation`, `Avg_Trend_Speed`, `Avg_Trend_Acceleration`
  - 특히 Speed·Acceleration은 타겟의 **미분값** → 직접 누수
- 🟡 **중복·저정보**: `DOI_freq`(−0.999 중복), `Paper_Age_Days`(중복), `Category_Similarity`(저정보)

### 분야 지배효과(confound) 보정

- 원본 `Category_Covariance`는 타겟과 corr **0.817** → 대부분 "AI/ML이냐"라는 분야 정체성
- **Option B** 적용: `Cov / 분야평균Cov` → corr **0.276**으로 완화
- ⚠️ 이전 json의 "0.092"는 재현 불가 → **0.276**이 정직한 값

## 📈 최종 결과

| 모델 | 정확도 | F1(macro) |
|------|--------|-----------|
| **DecisionTree** | **0.812** | **0.809** |
| RandomForest | 0.811 | 0.809 |
| KNN (k=11) | 0.683 | 0.675 |
| LogisticReg | 0.589 | 0.576 |

### 핵심 통찰

- **분야특징 기여도**: 빼면 F1 **0.48**, 넣으면 **0.81**
  → "분야 트렌드"는 주로 **분야 동조성**으로 결정, 논문 고유 특징은 보조
- **누수 없음**: F1이 1.0 근처가 아니라 0.81 → 정직한 일반화 성능
- 비선형 모델(트리)이 선형(로지스틱)을 크게 앞섬 → 관계가 비선형

## 💡 적용한 수업 개념 (복습용)

- **정규화·표준화**: 분산 큰 변수의 독점 방지 (PCA 데모로 확인)
